Step 2: The Agent "Brain"

Core Decision Logic

from langchain_openai.chat_models import ChatOpenAI
from langchain.schema import SystemMessage, HumanMessage, AIMessage
def call_model(state):
    msgs = state["messages"]
    order = state.get("order", {"order_id": "UNKNOWN"})
    # System prompt tells the model exactly what to do
    prompt = f'''You are an ecommerce support agent.
    ORDER ID: {order['order_id']}
    If the customer asks to cancel, call cancel_order(order_id) 
    and then send a simple confirmation.
    Otherwise, just respond normally.'''
    
    full = [SystemMessage(prompt)] + msgs
    
    # 1st LLM pass: decides whether to call our tool
    first = ChatOpenAI(model="gpt-4", temperature=0)(full)
    out = [first]

Step 3: Tool Execution Logic

Handling Tool Calls

    if getattr(first, "tool_calls", None):
        # run the cancel_order tool
        tc = first.tool_calls[0]
        result = cancel_order(**tc["args"])
        out.append(ToolMessage(content=result, tool_call_id=tc["id"]))

        # 2nd LLM pass: generate the final confirmation text
        second = ChatOpenAI(model="gpt-4", temperature=0)(full + out)
        out.append(second)

    return {"messages": out}

Two-step process: Execute tool ‚Üí Generate response

Step 4: Building the State Graph

Wiring It All Together

from langgraph.graph import StateGraph

# Wire it all up in a StateGraph
def construct_graph():
    g = StateGraph({"order": None, "messages": []})
    g.add_node("assistant", call_model)
    g.set_entry_point("assistant")
    return g.compile()

graph = construct_graph()

StateGraph manages conversation flow and state

Design Trade-offs

Balancing Performance, Scalability, Reliability & Cost

Strategic decisions that significantly impact real-world agent performance

Performance: Speed vs Accuracy

Prioritize Speed

  • When: Real-time environments
  • Examples: Autonomous vehicles, trading systems
  • Trade-off: May sacrifice precision for responsiveness
  • Solution: Fast approximation + refined follow-up

Prioritize Accuracy

  • When: High-stakes decisions
  • Examples: Medical diagnosis, legal analysis
  • Trade-off: Slower response times
  • Solution: Multi-stage validation processes

Hybrid Approach

Provide fast initial response, then refine with additional processing time and data

Scalability: Engineering for Growth

GPU Resource Management

GPUs are often the most expensive and limiting factor in scaling agent systems

Dynamic Allocation

Assign GPU resources based on real-time demand, reducing idle time

Elastic Provisioning

Auto-scale resources using cloud services or on-premises clusters

Priority Queuing

Give high-priority tasks immediate access while queuing less critical ones

Horizontal Scaling

  • Add more GPU nodes to handle workload
  • Cluster setup for high-volume tasks
  • Distributed inference across multiple GPUs

Hybrid Cloud Approach

  • On-premises + cloud GPU resources
  • Burst scaling during peak demand
  • Cost optimization through dynamic allocation

Reliability & Cost Considerations

Reliability Requirements

  • Fault Tolerance: Graceful error handling and recovery
  • Consistency: Reliable performance across scenarios
  • Testing: Extensive edge case validation
  • Monitoring: Continuous performance oversight

Cost Optimization

  • Development: Team expertise, testing infrastructure
  • Operational: Compute power, data storage, bandwidth
  • Strategies: Lean models, cloud resources, open source tools
  • ROI Balance: Cost justified by delivered value

Architecture Design Patterns

Single-Agent vs Multi-Agent Systems

Choosing the right structural pattern sets the stage for scalability and resilience

Single-Agent vs Multi-Agent Architectures

Single-Agent Architecture

  • Best for: Well-defined, narrow tasks
  • Advantages: Simple design, easy deployment
  • Examples: Basic chatbots, code generation
  • Scalability: Limited to single entity capabilities

Multi-Agent Architecture

  • Best for: Complex, distributed problems
  • Advantages: Specialization, parallelism, resilience
  • Examples: Financial trading, research platforms
  • Challenges: Coordination complexity, higher token usage

Multi-Agent Benefits

  • Collaboration: Specialized agents for different tasks
  • Parallelism: Simultaneous execution of multiple tasks
  • Redundancy: Failure in one agent doesn't compromise the system
  • Scalability: Add agents to handle increasing workload

    Context Engineering is the systematic discipline of designing complete informational environments for AI models. Unlike traditional prompt engineering that focuses on query phrasing, it encompasses:

    • System prompts: Foundational operational parameters
    • External data: Retrieved documents and tool outputs
    • Implicit context: User identity, history, environmental state

    Step 1: LangGraph State & Setup

    State Schema & LLM Initialization
    
from langgraph.graph import StateGraph, END
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from typing import TypedDict
# Initialize the Language Model
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
# Define the State schema
class ChainState(TypedDict):
    text_input: str         # Raw input text
    specifications: str     # Extracted specifications
    final_json: str        # Final JSON output

    State carries data through sequential processing nodes

    Step 2: LangGraph Node Functions

    Extract and Transform Node Implementations
    
def extract_specs(state: ChainState) -> ChainState:
    """Extract technical specifications from raw text"""
    prompt = ChatPromptTemplate.from_template(
        "Extract the technical specifications from the following text:\n\n{text_input}"
    )
    chain = prompt | llm | StrOutputParser()
    result = chain.invoke({"text_input": state["text_input"]})
    return {"specifications": result}
def transform_to_json(state: ChainState) -> ChainState:
    """Transform extracted specs into structured JSON"""
    prompt = ChatPromptTemplate.from_template(
        "Transform the following specifications into a JSON object with 'cpu', 'memory', and 'storage' as keys:\n\n{specifications}"
    )
    chain = prompt | llm | StrOutputParser()
    result = chain.invoke({"specifications": state["specifications"]})
    return {"final_json": result}

    Each node function processes state and returns updates

    Step 3: Graph Construction & Execution

    Build and Run the Chain Workflow
    
# Build the sequential workflow graph
workflow = StateGraph(ChainState)
workflow.add_node("extract", extract_specs)
workflow.add_node("transform", transform_to_json)
# Define the execution flow
workflow.add_edge("extract", "transform")
workflow.add_edge("transform", END)
workflow.set_entry_point("extract")
# Compile and execute the chain
chain_system = workflow.compile()
# Usage example
result = chain_system.invoke({
    "text_input": "The new laptop model features a 3.5 GHz octa-core processor, 16GB of RAM, and a 1TB NVMe SSD."
})
print("Final JSON Output:", result["final_json"])

    LangGraph provides stateful execution with explicit node transitions

    Flight API with Error Handling
    
@tool
def amadeus_flight_search(origin: str, destination: str, departure_date: str) -> Dict[str, Any]:
    """Search flights using Amadeus API"""
    try:
        # Get access token
        auth_url = "https://test.api.amadeus.com/v1/security/oauth2/token"
        auth_data = {
            "grant_type": "client_credentials",
            "client_id": AMADEUS_API_KEY,
            "client_secret": AMADEUS_SECRET
        }
        auth_response = requests.post(auth_url, data=auth_data)
        auth_response.raise_for_status()
        token = auth_response.json()["access_token"]

        # Search flights
        search_url = "https://test.api.amadeus.com/v2/shopping/flight-offers"
        headers = {"Authorization": f"Bearer {token}"}
        params = {
            "originLocationCode": origin,
            "destinationLocationCode": destination,
            "departureDate": departure_date,
            "adults": 1
        }

        response = requests.get(search_url, headers=headers, params=params)
        response.raise_for_status()

        return {
            "status": "success",
            "data": response.json()["data"][:5],  # Top 5 results
            "currency": response.json()["meta"]["currency"]
        }
    except requests.exceptions.RequestException as e:
        return {"status": "error", "message": f"Flight search failed: {str(e)}"}
    except KeyError as e:
        return {"status": "error", "message": f"Missing API response field: {str(e)}"}

    Step 3: Booking.com Hotel Search Tool

    Hotel API with Error Handling
    
@tool
def booking_hotel_search(city: str, checkin_date: str, checkout_date: str) -> Dict[str, Any]:
    """Search hotels using Booking.com API"""
    try:
        url = "https://booking-com.p.rapidapi.com/v1/hotels/search"
        headers = {
            "X-RapidAPI-Key": BOOKING_API_KEY,
            "X-RapidAPI-Host": "booking-com.p.rapidapi.com"
        }
        params = {
            "query": city,
            "checkin_date": checkin_date,
            "checkout_date": checkout_date,
            "adults_number": 1,
            "room_number": 1,
            "filter_by_currency": "USD",
            "order_by": "popularity"
        }

        response = requests.get(url, headers=headers, params=params)
        response.raise_for_status()
        data = response.json()

        hotels = []
        for hotel in data.get("result", [])[:5]:  # Top 5 hotels
            hotels.append({
                "name": hotel.get("hotel_name"),
                "price": hotel.get("min_total_price"),
                "rating": hotel.get("review_score"),
                "url": hotel.get("url")
            })
        return {"status": "success", "hotels": hotels}
    except requests.exceptions.RequestException as e:
        return {"status": "error", "message": f"Hotel search failed: {str(e)}"}
    except (KeyError, TypeError) as e:
        return {"status": "error", "message": f"Hotel data parsing error: {str(e)}"}

@tool
def eventbrite_search(city: str, start_date: str, category: str = None) -> Dict[str, Any]:
    """Search events using Eventbrite API"""
    try:
        url = "https://www.eventbriteapi.com/v3/events/search/"
        headers = {"Authorization": f"Bearer {EVENTBRITE_TOKEN}"}
        params = {"location.address": city, "start_date.range_start": f"{start_date}T00:00:00", "sort_by": "best"}
        if category: params["categories"] = category
        response = requests.get(url, headers=headers, params=params)
        response.raise_for_status()
        events = [{"name": e["name"]["text"], "url": e["url"], "start_time": e["start"]["local"]}
                 for e in response.json()["events"][:5]]
        return {"status": "success", "events": events}
    except Exception as e:
        return {"status": "error", "message": f"Events search failed: {str(e)}"}

def critic_agent(state: ReflectionState) -> ReflectionState:

    evaluation_criteria = {

    critique = generate_critique(evaluation_criteria, state["current_output"])

def should_continue_reflection(state: ReflectionState) -> str:
    """Determine if reflection should continue"""
    if state["approved"]:
        return "complete"
    elif state["iteration_count"] >= state["max_iterations"]:
        return "complete"  # Max iterations reached
    else:
        return "continue"

class PromptEvolution:
    def __init__(self, config: Dict[str, Any]):
        self.config = config
        self.population = []
        self.fitness_scores = {}
        self.generation_history = []

    def generate_mutations(self, prompt: str, num_mutations: int = 3) -> List[str]:
        """Generate prompt mutations using LLM"""
        mutation_prompt = f"""
        Improve this prompt for better performance: "{prompt}"
        Generate {num_mutations} variations that are:
        1. More specific and clear
        2. Better structured
        3. More likely to produce accurate results
        """
        # LLM call to generate mutations
        mutations = self.llm_generate(mutation_prompt)
        return mutations

    def evaluate_prompt(self, prompt: str, dataset: List[Dict]) -> float:
        """Evaluate prompt performance on dataset"""
        correct = 0
        total = len(dataset)
        for sample in dataset:
            response = self.llm_query(prompt, sample["input"])
            if self.is_correct(response, sample["expected"]):
                correct += 1
        return correct / total

    Core evolution class with mutation generation and fitness evaluation methods

    Step 2: Evolutionary Process

    MAP-Elites Algorithm Implementation
    
def evolve_prompts(self, baseline_prompt: str, dataset: List[Dict]) -> str:
    """Main evolutionary optimization loop"""
    # Initialize population with baseline prompt
    self.population = [baseline_prompt]
    best_prompt = baseline_prompt
    best_score = 0.0

    for generation in range(self.config["evolution"]["generations"]):
        print(f"Generation {generation + 1}")

        # Generate mutations using LLM
        new_prompts = []
        for prompt in self.population:
            mutations = self.generate_mutations(prompt)
            new_prompts.extend(mutations)

        # Evaluate all prompts
        for prompt in new_prompts:
            score = self.evaluate_prompt(prompt, dataset)
            self.fitness_scores[prompt] = score

            if score > best_score:
                best_score = score
                best_prompt = prompt

        # Selection for next generation
        self.population = self.select_best_prompts(new_prompts)
        print(f"Best score: {best_score:.3f}")

    return best_prompt

                            request: str,
                            ai_response: str,

        # Low confidence check
        if confidence < self.escalation_rules["confidence_threshold"]:

        # High stakes domain check
        if domain in self.escalation_rules["high_stakes_domains"]:
            escalation_reasons.append(f"High-stakes domain: {domain}")

        # Sensitive topic check
        if self._contains_sensitive_topics(request):
            escalation_reasons.append("Contains sensitive topics")

        if escalation_reasons:
            return EscalationRequest(
                request_id=self._generate_request_id(),
                user_query=request,
                ai_response=ai_response,
                confidence_score=confidence,
                risk_level=risk_level,
                domain=domain,
                escalation_reason="; ".join(escalation_reasons),
                suggested_experts=self.expert_registry.get(domain, []),
                context=context
            )

        return None

    async def geocode_location(self, location_name: str):
        """Convert city name to coordinates using OpenWeatherMap API"""
        params = {
            "q": location_name,
            "limit": 1,
            "appid": self.api_key
        }

        async with httpx.AsyncClient() as client:
            response = await client.get(self.base_url, params=params)
            response.raise_for_status()
            data = response.json()

            if data:
                result = data[0]
                return {
                    "name": result["name"],
                    "country": result["country"],
                    "lat": result["lat"],
                    "lon": result["lon"],
                    "state": result.get("state", "")
                }
            else:
                raise ValueError(f"Location '{location_name}' not found")

class WeatherAgent:
    """Weather data using OpenWeatherMap One Call API 3.0"""

    def __init__(self):
        self.api_key = os.getenv("OPENWEATHERMAP_API_KEY")
        self.base_url = "https://api.openweathermap.org/data/3.0/onecall"

    async def get_weather_data(self, lat: float, lon: float):
        """Get current weather and forecasts using coordinates"""
        params = {
            "lat": lat,
            "lon": lon,
            "appid": self.api_key,
            "units": "metric",
            "exclude": "minutely,alerts"  # Focus on current + hourly + daily
        }

        async with httpx.AsyncClient() as client:
            response = await client.get(self.base_url, params=params)
            response.raise_for_status()
            data = response.json()

            current = data["current"]
            return {
                "current": {
                    "temperature": current["temp"],
                    "feels_like": current["feels_like"],
                    "humidity": current["humidity"],
                    "pressure": current["pressure"],
                    "description": current["weather"][0]["description"],
                    "icon": current["weather"][0]["icon"]
                },
                "hourly_forecast": data["hourly"][:24],  # Next 24 hours
                "daily_forecast": data["daily"][:7],     # Next 7 days
                "timezone": data["timezone"]
            }

weather_agent = WeatherAgent()

async def a2a_geocoding_endpoint(request):
    """A2A endpoint for geocoding agent on port 8002"""
    data = await request.json()

    if data.get("method") == "sendTask":
        params = data["params"]
        location_text = params["message"]["parts"][0]["text"]

        try:
            result = await geocoding_agent.geocode_location(location_text)
            return JSONResponse({
                "jsonrpc": "2.0",
                "id": data["id"],
                "result": {
                    "task_id": params["id"],
                    "status": "completed",
                    "output": result
                }
            })
        except Exception as e:
            return JSONResponse({
                "jsonrpc": "2.0",
                "id": data["id"],
                "error": {"code": -1, "message": str(e)}
            })

async def a2a_weather_endpoint(request):

    A2A vs MCP Protocols

    A2A (Agent-to-Agent)

    • Purpose: Inter-agent communication and coordination
    • Scope: Multi-agent system orchestration
    • Protocol: HTTP-based with JSON-RPC 2.0
    • Discovery: Agent Cards and registries
    • Use Case: Cross-framework agent collaboration

    MCP (Model Context Protocol)

    • Purpose: Model-to-resource communication
    • Scope: External data and tool access
    • Protocol: Standardized context interface
    • Discovery: Resource and tool catalogs
    • Use Case: LLM external system integration

    Complementary Protocols

    A2A enables agent-to-agent collaboration while MCP handles agent-to-resource communication. Together they provide comprehensive connectivity for multi-agent systems.
